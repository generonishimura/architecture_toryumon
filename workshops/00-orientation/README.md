# 第0回：オリエンテーション

> 全体像を共有し、学習の準備を整える

## 基本情報

| 項目 | 内容 |
|------|------|
| 所要時間 | 1.5時間 |
| 前提知識 | TypeScript基本文法、Git基本操作 |
| 到達目標 | ワークショップの全体像を理解し、学習環境が整っている状態 |

## アジェンダ

### 1. イントロダクション（15分）

- ワークショップ「アーキテクチャ登竜門」の趣旨説明
- 「アーキテクチャ」とは何か — ソフトウェアにおける設計判断の集合
- 本ワークショップで扱う範囲と扱わない範囲

### 2. 参加者自己紹介（15分）

- 名前、チーム、主な担当領域
- 現在のアーキテクチャに関する悩み・学びたいこと
- （事前アンケート結果があれば共有）

### 3. 全体ロードマップの紹介（20分）

- 9つのテーマの概要と順序の意図
- フェーズ構成（基礎 → 設計パターン → 分散システム）
- 各テーマがどうつながるかの関連図

### 4. 学習の進め方（15分）

- 各回の構成（講義 + ディスカッション + Q&A）
- 課題の取り組み方とフィードバックの仕組み
- 質問・議論の推奨（「分からない」を歓迎する文化）

### 5. 環境構築（20分）

- Node.js / TypeScript の開発環境確認
- リポジトリのクローンとセットアップ
- エディタ（VS Code推奨）の設定

### 6. ウォームアップ：設計について考える（15分）

以下の問いについて、グループで議論する。

- 「良いコード」と「良い設計」の違いは何か？
- 設計で一番困った経験は？
- なぜアーキテクチャを学ぶ必要があるのか？
