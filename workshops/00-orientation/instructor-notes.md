# 第0回 オリエンテーション — 講師補助資料

## この回の役割

初回は「学びの場の空気づくり」が最優先。内容の深さより、参加者が安心して発言できる雰囲気を作ることに注力する。

## 講師の事前準備チェックリスト

- [ ] 全11回のカリキュラムを通読し、質問に概要レベルで答えられる状態にする
- [ ] 事前アンケート結果を集計し、参加者の関心テーマ・不安な点を把握する
- [ ] 環境構築手順を自分で一通り試し、つまずきポイントを把握する
- [ ] 自己紹介で場を温めるための自分のエピソード（設計で失敗した話など）を用意する

## 時間配分の注意

| パート | 配分 | 注意点 |
|--------|------|--------|
| イントロダクション | 15分 | 長くなりがち。「詳しくは各回で」と切り上げる |
| 自己紹介 | 15分 | 人数が多い場合は1人1分厳守。タイマー推奨 |
| ロードマップ紹介 | 20分 | 全体図を見せつつ「全部覚えなくていい」と安心感を |
| 学習の進め方 | 15分 | 課題の負荷感を正直に伝える |
| 環境構築 | 20分 | **最も遅延リスクが高い。** 事前にトラブルシューティングリストを準備 |
| ウォームアップ | 15分 | ここが一番大事。全員が1回は発言する状態を目指す |

## 想定される質問と回答例

| 質問 | 回答の方向性 |
|------|-------------|
| 「ついていけるか不安です」 | 「このワークショップは『知らない』が前提。分からないことを言える場にしたい」 |
| 「課題にどれくらい時間がかかりますか」 | 「復習問題15分＋コード課題30〜60分が目安。発展課題は任意なので無理しなくてOK」 |
| 「自分のプロジェクトに直接役立ちますか」 | 「各回で自分のプロジェクトに当てはめて考える時間を設けるので、実務との接点は意識して進める」 |
| 「○○回のテーマは経験があるのですが」 | 「経験者にはディスカッションでリード役をお願いしたい。教えることで理解がさらに深まる」 |

## ウォームアップの進め方のコツ

3つの問いを全部やろうとしない。1つ選んで深く議論するほうが効果的。

**おすすめの進め方:**
1. 「良いコードと良い設計の違いは？」を全体に投げかける（1分）
2. 隣の人と2人で話す（3分）
3. 2〜3組に発表してもらう（5分）
4. 講師が「これからの11回で、その『良い設計』を体系的に学びます」とつなげる（1分）

## 環境構築トラブルシューティング

| トラブル | 対処 |
|---------|------|
| Node.js のバージョンが古い | nvm での切り替え手順を案内 |
| npm install が企業プロキシで失敗 | `.npmrc` のプロキシ設定を案内 |
| VS Code の拡張機能が入らない | オフラインインストール手順を準備 |
| Windows で ts-node が動かない | PowerShell の実行ポリシー変更を案内 |

## 次回（第1回）への橋渡し

終了時に必ず伝えること:
- 「次回はオブジェクト指向。SOLID原則を中心に扱います」
- 「事前に知っている必要はないが、『カプセル化・継承・ポリモーフィズム』という言葉だけ頭に入れてきてください」
- 「自分が書いたコードで『設計が良い/悪い』と感じた経験を1つ思い出しておいてください」
